Modulo 3 - Front React - Sprint 4

Kenzie Hub

Crie uma aplicacao com login, cadastro visual, rota protegida e dashboard onde o usuario controla suas tecnologias.

React Routerformularioszod opcionalcontext opcional

Dependencias principais

  • react e react-dom: base da aplicacao.
  • vite e @vitejs/plugin-react: criacao e execucao do projeto.
  • react-router-dom: rotas publicas e protegidas.
  • react-hook-form e zod: validacao dos formularios.
  • @hookform/resolvers: liga zod ao react-hook-form.
  • axios ou fetch: login na API.
  • react-modal e react-toastify: opcionais para modal e mensagens.

Referencia visual

Use este link como guia de layout: Kenzie Hub publicado.

API que deve usar

Use DummyJSON Auth para login. Para as tecnologias, use estado local e localStorage; assim o aluno nao depende de uma API complexa para CRUD.

POST https://dummyjson.com/auth/login
body:
{
  "username": "emilys",
  "password": "emilyspass"
}

Depois do login, salve accessToken e dados do usuario no localStorage.

Ordem de montagem

  1. Crie o projeto e configure as paginas Login, Register e Dashboard.
  2. Configure o roteamento. Login e cadastro sao publicos; dashboard e protegido.
  3. Crie uma funcao para verificar se existe token salvo no localStorage.
  4. Monte o formulario de login com validacao de campos vazios.
  5. No login, envie username e password para DummyJSON Auth.
  6. Se der certo, salve accessToken e dados do usuario; depois navegue para dashboard.
  7. Se der errado, mostre erro claro e mantenha o usuario na tela de login.
  8. Monte o dashboard com dados do usuario, botao logout e lista de tecnologias.
  9. Crie modal ou formulario para adicionar tecnologia com nome e nivel.
  10. Implemente editar nivel, excluir tecnologia e persistir tudo no localStorage.
  11. Finalize logout limpando usuario, token e redirecionando para login.

Regras de aceite

  • Usuario sem token nao pode entrar no dashboard.
  • Formulario de login precisa validar campos vazios.
  • Tecnologia precisa ter nome e nivel.
  • Nivel deve ser uma lista fechada: Iniciante, Intermediario, Avancado.
  • As tecnologias devem permanecer apos atualizar a pagina.
  • Use modal ou pagina separada para criar/editar tecnologias.
