Modulo 3 - Front React - Sprint 3

Hamburgueria

Crie uma vitrine de produtos com busca e carrinho. Nao existe layout oficial aqui: faca uma tela limpa, responsiva e facil de usar.

ReactAPI publicauseEffectlocalStorage

Dependencias principais

  • react e react-dom: base da aplicacao.
  • vite e @vitejs/plugin-react: criacao e execucao do projeto.
  • axios ou fetch: consumo da API de produtos.
  • react-toastify: opcional, para mensagens de erro e sucesso.
  • styled-components: opcional, para organizar o CSS por componente.

API que deve usar

Use DummyJSON Products no lugar da API antiga. Ela e simples, gratuita e retorna produtos com nome, preco, categoria e imagem.

GET https://dummyjson.com/products
GET https://dummyjson.com/products/search?q=phone
GET https://dummyjson.com/products/category-list

Campos principais: id, title, category, price, thumbnail.

Ordem de montagem

  1. Crie o projeto React e monte o header com nome, busca e botao do carrinho.
  2. Crie no App os estados products, cart, search, loading e error.
  3. No useEffect inicial, busque https://dummyjson.com/products.
  4. Enquanto carrega, mostre loading. Se der erro, mostre uma mensagem simples.
  5. Renderize cards com thumbnail, title, category, price e botao adicionar.
  6. Implemente a busca. Primeiro pode filtrar localmente; depois, se quiser, use /products/search?q=texto.
  7. Ao adicionar produto, verifique se ele ja existe no carrinho. Se existir, aumente quantity.
  8. Crie o carrinho com lista de itens, quantidade, remover item, limpar tudo e total.
  9. Salve o carrinho no localStorage sempre que ele mudar.
  10. Finalize responsivo e garanta que produto, busca e carrinho sejam usaveis no celular.

Regras de aceite

  • A aplicacao precisa consumir uma API publica real.
  • Erro de API deve mostrar uma mensagem amigavel, nao tela quebrada.
  • O carrinho precisa mostrar quantidade e total em reais.
  • A busca precisa informar quando nao encontrou nenhum produto.
  • O layout precisa funcionar no celular e no desktop.

Componentes sugeridos

  • Header: logo, busca e quantidade de itens.
  • ProductList: recebe produtos e renderiza cards.
  • ProductCard: card individual.
  • Cart: lista itens, total e limpar carrinho.
  • CartItem: item individual com remover.
