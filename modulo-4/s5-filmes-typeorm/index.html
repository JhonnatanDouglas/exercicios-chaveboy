Locadora de Filmes com TypeORM

Recrie a API de filmes usando TypeORM, migrations, zod e paginacao. O foco e trocar SQL manual por entidade e repositorio.

TypeORMmigrationszodpaginacao

Dependencias principais

  • express e express-async-errors: API e erros async.
  • typescript e ts-node-dev: desenvolvimento em TypeScript.
  • typeorm, reflect-metadata e pg: ORM e PostgreSQL.
  • dotenv: variaveis de ambiente.
  • zod: validacao dos dados.
  • sqlite3: banco em memoria para testes, se necessario.
  • jest, supertest e ts-jest: testes de integracao, se forem exigidos.

Entidade Movie

  • id: inteiro, primary generated.
  • name: varchar 50, unico, obrigatorio.
  • description: text, opcional, default null.
  • duration: inteiro, obrigatorio.
  • price: inteiro, obrigatorio.

Endpoints

MetodoRotaFuncao
POST/moviesCriar filme
GET/moviesListar com paginacao, sort e order
PATCH/movies/:idAtualizar filme
DELETE/movies/:idDeletar filme

Paginacao

A rota GET recebe page, perPage, sort e order.

  • page: padrao 1.
  • perPage: padrao 5, maximo 5.
  • sort: aceita price ou duration. Sem sort, ordene por id.
  • order: aceita asc ou desc. Padrao asc.
{
  "prevPage": null,
  "nextPage": "http://localhost:3000/movies?page=2&perPage=5",
  "count": 12,
  "data": []
}

Ordem de montagem

  1. Configure TypeScript com decorators e importe reflect-metadata.
  2. Configure data-source.ts com PostgreSQL em dev e SQLite em teste, se quiser.
  3. Crie entidade Movie e exporte por entities/index.ts.
  4. Crie e rode a migration inicial da tabela movies.
  5. Crie schemas zod para POST e PATCH; chaves extras devem ser ignoradas.
  6. Crie rota POST com validacao de body e nome duplicado.
  7. Crie middleware de paginacao para tratar page, perPage, sort e order.
  8. Crie GET geral retornando prevPage, nextPage, count e data.
  9. Crie middleware de id inexistente para PATCH/DELETE.
  10. Finalize PATCH parcial e DELETE usando repository do TypeORM.

Regras de aceite

  • POST deve exigir name, duration e price.
  • description e opcional e deve salvar como null se nao vier.
  • name deve ser unico.
  • POST e PATCH devem ignorar id enviado pelo cliente.
  • GET deve paginar e limitar perPage a 5.
  • DELETE deve retornar 204 quando remover com sucesso.
