Modulo 4 - Backend Express - Sprint 1

API de Mercado Autonomo

Crie uma API REST em Express e TypeScript para gerenciar produtos de um mercado. Nesta sprint, o banco pode ser um array em memoria.

NodeExpressTypeScriptarray em memoria

Dependencias principais

  • express: criar servidor e rotas.
  • typescript: tipagem do projeto.
  • ts-node-dev: rodar o servidor em desenvolvimento.
  • @types/express: tipos do Express.

Modelo do produto

{
  "id": 1,
  "name": "Queijo",
  "price": 10,
  "section": "food",
  "weight": 30,
  "calories": 300,
  "expirationDate": "2026-06-01T00:00:00.000Z"
}

Produtos de food usam weight, calories e expirationDate. Produtos de cleaning usam weight e expirationDate. O vencimento pode ser gerado automaticamente para 1 ano no futuro.

Endpoints

MetodoRotaFuncao
POST/productsCriar produto
GET/productsListar todos e retornar total
GET/products/:idBuscar um produto
PATCH/products/:idAtualizar parcialmente
DELETE/products/:idRemover produto

Ordem de montagem

  1. Inicie o projeto Node, instale dependencias e configure tsconfig.json.
  2. Crie src/app.ts, configure Express e express.json().
  3. Crie interfaces.ts com os tipos de produto.
  4. Crie database.ts exportando o array products.
  5. Crie a rota POST e valide campos obrigatorios antes de salvar.
  6. Crie middleware para bloquear nome duplicado.
  7. Crie GET geral retornando { total, products }.
  8. Crie middleware para validar id em GET/PATCH/DELETE.
  9. Crie GET por id, PATCH parcial e DELETE.
  10. Revise status codes e mensagens de erro.

Regras de aceite

  • name, price, section e weight sao obrigatorios.
  • section aceita apenas food ou cleaning.
  • Dois produtos nao podem ter o mesmo nome.
  • Id inexistente deve responder 404.
  • A rota GET geral deve responder { total, products }.
