Modulo 4 - Backend Express - Sprint 2

Locadora de Filmes com PostgreSQL

Crie uma API REST para cadastrar, listar, atualizar e deletar filmes usando PostgreSQL, Express e TypeScript.

ExpressPostgreSQLpgpg-format

Dependencias principais

  • express: servidor e rotas.
  • typescript e ts-node-dev: desenvolvimento em TypeScript.
  • pg: conexao e queries no PostgreSQL.
  • pg-format: montagem segura de updates dinamicos.
  • dotenv: leitura do DATABASE_URL.
  • @types/express, @types/pg e @types/pg-format: tipos.

Tabela

CREATE TABLE movies (
  id SERIAL PRIMARY KEY,
  name VARCHAR(50) NOT NULL UNIQUE,
  category VARCHAR(20) NOT NULL,
  duration INTEGER NOT NULL,
  price INTEGER NOT NULL
);

Endpoints

MetodoRotaFuncao
POST/moviesCriar filme
GET/moviesListar filmes; aceitar ?category=
GET/movies/:idBuscar por id
PATCH/movies/:idAtualizar parcialmente
DELETE/movies/:idDeletar

Ordem de montagem

  1. Configure projeto, scripts e .env com DATABASE_URL.
  2. Crie sql/create_table.sql com a tabela movies.
  3. Crie database.ts com Pool do pg e funcao para iniciar tabela.
  4. Crie app.ts com Express e suba o banco antes do servidor.
  5. Crie POST /movies usando query parametrizada.
  6. Crie middleware para bloquear name duplicado.
  7. Crie GET geral e filtro por ?category=.
  8. Crie middleware para id inexistente.
  9. Crie GET por id, PATCH dinamico e DELETE.
  10. Teste todos os fluxos no Insomnia ou Thunder Client.

Regras de aceite

  • name, category, duration e price sao obrigatorios no POST.
  • name deve ser unico.
  • Id inexistente retorna 404 com mensagem clara.
  • PATCH nao deve exigir todos os campos.
  • GET /movies?category=acao deve filtrar; se nao encontrar, pode retornar lista vazia.
