Modulo 4 - Backend Express - Sprint 4

Usuarios, Cursos e Permissao Admin

Crie uma API com cadastro de usuarios, login JWT, cursos e matriculas. Algumas rotas sao publicas, outras exigem token e admin.

JWTbcryptzodPostgreSQL

Dependencias principais

  • express e express-async-errors: API e erros async.
  • typescript e ts-node-dev: desenvolvimento em TypeScript.
  • pg e pg-format: PostgreSQL e queries dinamicas.
  • dotenv: variaveis de ambiente.
  • zod: validacao e serializacao.
  • bcryptjs: hash e comparacao de senha.
  • jsonwebtoken: login com token JWT.
  • jest, supertest e ts-jest: testes de integracao, se forem exigidos.

Tabelas

  • users: id, name, email unico, password hash, admin boolean default false.
  • courses: id, name, description.
  • userCourses: id, active default true, userId, courseId.

Endpoints

MetodoRotaProtecaoFuncao
POST/usersPublicaCriar usuario
POST/loginPublicaGerar token
GET/usersAdminListar usuarios
GET/users/:id/coursesAdminCursos de um usuario
POST/coursesAdminCriar curso
GET/coursesPublicaListar cursos
POST/courses/:courseId/users/:userIdAdminMatricular usuario
GET/courses/:id/usersAdminUsuarios de um curso
DELETE/courses/:courseId/users/:userIdAdminDesativar matricula

Ordem de montagem

  1. Crie SQL das tres tabelas com chaves estrangeiras e ON DELETE CASCADE.
  2. Configure conexao, .env, SECRET_KEY e middleware global de erros.
  3. Crie schemas zod para usuario, login e curso.
  4. Implemente POST /users: validar body, bloquear email duplicado, gerar hash e esconder senha.
  5. Implemente POST /login: buscar usuario, comparar senha e gerar JWT.
  6. Crie verifyToken para ler Bearer token e salvar dados do usuario logado.
  7. Crie validateAdmin e proteja rotas administrativas.
  8. Implemente GET /users e GET /users/:id/courses.
  9. Implemente POST/GET /courses.
  10. Implemente matricula, listagem de usuarios no curso e desativacao com active = false.

Regras de aceite

  • Nunca retornar password nas respostas.
  • Email precisa ser unico e valido.
  • Token ausente, invalido ou mal formatado deve retornar 401.
  • Usuario comum nao pode acessar rotas admin.
  • Uma matricula ja existente nao deve ser duplicada.
